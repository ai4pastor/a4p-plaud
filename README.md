# A4P plaud

Plaud 녹음·전사록을 옵시디언으로 가져와 사이드패널에서 스트리밍 재생하고, 템플릿·성경 wikilink 자동화로 PKM과 통합하는 옵시디언 플러그인입니다.

> 📦 ai4pastor 강의 수강생 배포용. **데스크탑 전용** (모바일 미지원).

## 주요 기능

- **사이드패널**에서 Plaud 녹음 검색·필터·정렬, 한눈에 보는 카드 리스트
- **노트 임포트** (수동): 트랜스크립트·AI 요약을 메타와 함께 vault에 저장
- **스트리밍 플레이어**: vault에 mp3 저장하지 않고 재생 직전 URL 갱신 (vault 무거워지지 않음)
- **임포트 템플릿** + **Templater 연동**: 사용자 템플릿 스크립트로 frontmatter 자동 채움
- **성경 구절 자동 wikilink** 변환 (한국어 66권, `요한복음 3:16` → `[[요3_16]]`)
- **활성 노트 ↔ 카드 시각 동기**: 노트 열면 사이드패널 카드 자동 강조·스크롤

## 설치 (BRAT)

1. 옵시디언에 **BRAT**(Beta Reviewers Auto-update Tester) 플러그인 설치·활성화
2. 명령 팔레트 → **"BRAT: Add a beta plugin for testing"**
3. 입력: `ai4pastor/a4p-plaud`
4. 설치 완료 후 설정 → 커뮤니티 플러그인에서 **A4P plaud** 활성화

## Plaud 토큰 발급

Plaud는 구글 등 OAuth 로그인을 사용하므로 옵시디언에서 직접 로그인할 수 없습니다. 브라우저에서 access_token을 **한 번만** 추출하면 약 300일간 사용할 수 있습니다.

### 방법 A — Network 탭 (권장)

1. 브라우저로 `https://app.plaud.ai` 로그인 (구글 로그인 그대로 OK)
2. 개발자 도구 열기 — Mac: `Cmd+Opt+I`, Windows: `F12`. 상단에서 **Network** 탭 선택
3. Plaud 웹에서 아무 작업 (예: 녹음 목록 새로고침, 녹음 하나 클릭). Network 패널에 요청이 흘러갑니다
4. 요청 중 아무거나 하나 클릭 → 우측 **Headers** → Request Headers 영역에서 `Authorization: Bearer ...` 줄을 찾기
5. `Bearer ` 다음의 매우 긴 문자열(보통 `eyJ`로 시작, 200자 이상) 전체 복사. **끝부분 공백/줄바꿈 들어가지 않게 주의**
6. 옵시디언 설정 → **A4P plaud** → 토큰 입력란 붙여넣고 **저장 및 검증**

### 방법 B — Application 탭 (대안)

1. 개발자 도구 → **Application** 탭 → 좌측 Storage → **Local Storage** → `https://app.plaud.ai` 선택
2. 키 목록에서 `access_token` (또는 token / accessToken) 값을 복사
3. 값이 `eyJ`로 시작하는 긴 문자열이면 그대로 사용. 아니면 방법 A로 진행

### 🌍 리전(서버 위치) 안내

Plaud는 사용자 위치에 따라 **미국(US) / 유럽(EU) / 일본·아시아(APNE1)** 등 여러 리전에 데이터가 저장됩니다. 본 플러그인은 토큰을 받은 뒤 자동으로 올바른 리전을 감지합니다 (필요 시 한 번 자동 재시도). 검증 후 "**리전: APNE1**"처럼 표시되면 정상입니다.

> ⚠️ **"Plaud 리전을 확인할 수 없습니다" 에러가 뜨면**
> 1. 같은 브라우저에서 한 번 더 토큰을 받아 시도
> 2. VPN 사용 중이면 끄고 다시 시도
> 3. 그래도 실패 시 개발자 콘솔(`Cmd+Opt+I`)에서 `[A4P Plaud] -302 redirect` 로그를 복사해 이슈로 보고해 주세요. 새 리전(중국·인도 등)이라면 후속 업데이트로 지원 추가

### 🔒 보안

입력한 토큰은 운영체제 키체인(`safeStorage`)으로 암호화되어 vault에 평문으로 저장되지 않습니다. 만료(약 300일 후) 시 같은 방법으로 새 토큰을 받아 다시 입력해 주세요.

## 사용법

### 사이드패널 열기

- 좌측 리본의 🎙 마이크 아이콘 클릭 또는 명령 팔레트 → **"Plaud 패널 열기"**

### 녹음 듣기

- 카드를 **클릭**하면 플레이어에 로드됩니다 (자동 재생 X)
- 플레이어의 **▶ 재생** 버튼으로 시작 — 첫 재생 시 mp3 URL을 받아옵니다
- 컨트롤: ⏪10초 / 10초⏩ / 1.0x · 1.5x · 2.0x 배속 / 시크바

### 노트 임포트

- 카드 우측 **📄 노트 보기** 클릭 → 모달에서 트랜스크립트 미리보기 + **노트로 가져오기**
- 임포트된 카드는 좌측 그린 보더 + **📄 노트 열기** (강조색)
- 기본 저장 폴더: `Plaud/` (설정에서 변경 가능)
- 노트 파일명: `{YYYY-MM-DD HHmm} {녹음 제목}.md`

### 임포트 템플릿 (선택)

설정 → 임포트 → **기본 임포트 템플릿**에 vault 내 .md 파일 경로 지정. 모달에서도 매번 변경 가능.

지원 변수 (단순 치환):

| 변수 | 값 |
|---|---|
| `{{plaud_id}}` | 녹음 고유 ID |
| `{{transcript}}` | 전체 트랜스크립트 |
| `{{summary}}` | AI 요약 (없으면 빈 문자열) |
| `{{filename}}` | 원본 파일명 |
| `{{date}}` | 녹음일 (YYYY-MM-DD HH:mm) |
| `{{duration}}` | 길이 (mm:ss) |
| `{{duration_seconds}}` | 길이 (초) |
| `{{region}}` | 데이터 리전 (us / eu / apne1 등) |
| `{{imported_at}}` | 임포트 시각 |

**Templater 연동**: Templater 플러그인이 설치되어 있으면, 사용자 템플릿 안의 `<% ... %>` 또는 `<%* ... _%>` 블록을 노트 생성 직후 자동 실행합니다. 사용자 스크립트가 `processFrontMatter`로 채운 값은 본 플러그인이 자동으로 보존합니다.

**옵션 X 모드**: 노트 본문(메타·요약·트랜스크립트)은 본 플러그인이 채우고, 사용자 템플릿은 본문 끝에 append됩니다. 사용자 템플릿 시작 부분의 frontmatter 블록은 자동 제거되므로 스크립트만 두셔도 됩니다.

### 성경 구절 자동 wikilink

활성 노트에서 명령 팔레트 → **"활성 노트의 성경 구절을 wikilink로 변환"**

| 입력 | 변환 |
|---|---|
| `요한복음 3장 16절` | `[[요3_16]]` |
| `요 3:16` / `요3:16` | `[[요3_16]]` |
| `시편 23편 1-6절` | `[[시23_1]] ~ [[시23_6]]` |
| `고린도전서 13:4~7` | `[[고전13_4]] ~ [[고전13_7]]` |

- 한국어 66권 책 이름·줄임표 모두 인식
- 코드 블록·기존 wikilink 안은 건드리지 않음
- frontmatter는 보존
- 실수 시 `Cmd+Z`로 되돌리기 가능

## 활성 노트 ↔ 카드 동기

- vault에서 plaud 노트(`plaud_id` frontmatter가 있는 노트)를 열면 사이드패널 카드가 자동 강조 + 그 위치로 스크롤
- 일반 노트로 전환해도 플레이어 상태는 유지

## 알려진 제한

- **데스크탑 전용** (옵시디언 모바일 미지원 — `safeStorage` 의존)
- **트랜스크립트 타임스탬프 없음**: Plaud transcript에 시간 정보가 없어 줄→오디오 점프 기능은 미지원
- **자동 동기화 없음**: 새 녹음을 자동으로 가져오지 않음. 사이드패널 ↻ 새로고침 또는 카드 ⓘ 클릭으로 수동 임포트
- **mp3 다운로드 없음** (기본): 재생 직전 URL을 새로 받아 스트리밍. 인터넷 필요 (오프라인 캐시는 추후 옵션)

## 트러블슈팅

**"토큰이 이미 만료되었습니다"** → 토큰 발급을 다시 진행해 주세요.

**"Plaud 리전을 확인할 수 없습니다"** → 콘솔(`Cmd+Opt+I`)에서 `[A4P Plaud] -302 redirect` 로그 확인 후 이슈로 보고해 주세요.

**Templater 분류 결과가 사라짐** → v0.1.0부터 해결됨. 그래도 발생하면 콘솔 로그 확인.

**오디오 재생이 안 됨** → 24h 후 presigned URL 만료. 자동 갱신되도록 처리되어 있지만, 안 되면 카드 다시 클릭.

## 개발

```bash
git clone https://github.com/ai4pastor/a4p-plaud.git
cd a4p-plaud
npm install
npm run dev   # esbuild watch
```

옵시디언 dev vault의 `.obsidian/plugins/a4p-plaud`로 심볼릭 링크.
